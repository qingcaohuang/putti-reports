## 一、记忆架构升级（对标 Claude Code 三层记忆 + autoDream）

### 现状
- 两层结构：`memory/YYYY-MM-DD.md`（日志）+ `MEMORY.md`（长期）
- AGENTS.md 里有"Memory Maintenance"指引，但没有自动化执行
- HEARTBEAT.md 当前为空，heartbeat 机制未启用

### 行动项

| # | 行动 | 难度 | 优先级 | 说明 |
|---|------|------|--------|------|
| 1.1 | 启用 heartbeat 记忆整理 | 低 | ⭐高 | 在 HEARTBEAT.md 中加入"每日记忆整理"任务，heartbeat 时自动回顾近 3 天日志，提炼有价值内容更新 MEMORY.md。类似 autoDream 的"睡眠整理"机制 |
| 1.2 | 引入中期记忆层 | 中 | ⭐中 | 新增 `memory/weekly-YYYY-WW.md` 周总结，介于日志和长期记忆之间。每周一 heartbeat 自动生成上周总结 |
| 1.3 | 记忆分类标签 | 低 | ⭐中 | 在 MEMORY.md 中按领域分区（技术决策、用户偏好、项目进展、教训），便于检索和维护 |
| 1.4 | 记忆过期清理 | 低 | ⭐低 | 超过 30 天的日志中，未被提炼到 MEMORY.md 的内容标记为可归档 |

## 二、上下文管理优化（对标四层上下文压缩）

### 现状
- 60%/75%/90% 三级预警机制（cron 每小时检查）
- 75% 强制写入 memory
- 但没有"按需加载"机制，session 启动时一次性加载所有文件

### 行动项

| # | 行动 | 难度 | 优先级 | 说明 |
|---|------|------|--------|------|
| 2.1 | Session 启动精简加载 | 中 | ⭐高 | 区分"必读"（SOUL.md、USER.md、今日 memory）和"按需读"（MEMORY.md 全文、历史 memory）。非主 session 不加载 MEMORY.md 已在执行 |
| 2.2 | 任务级上下文隔离 | 中 | ⭐中 | 复杂任务拆分到子 agent 执行，主 session 只保留任务摘要。ClawTeam 的 git worktree 隔离思路可直接复用 |
| 2.3 | memory_search 优先于全文读取 | 低 | ⭐高 | 需要历史信息时，先 memory_search 定位再 memory_get 精准读取，避免全文加载浪费 token。这个已经是规则，需要更严格执行 |

## 五、安全与反蒸馏（了解即可）

Claude Code 的三层反蒸馏机制（输出注入标记、请求指纹、行为模式检测）是商业防护措施，对我们没有直接应用价值，但提醒我们：
- 使用第三方 API 时，输出可能包含隐藏标记
- 不要用 AI 输出直接训练模型（法律风险）
